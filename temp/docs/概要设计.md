# 膳智膳食管理系统概要设计说明书

## 1. 引言

### 1.1 编写目的
本文档为"膳智"（DietWise）智能膳食管理系统提供全面的技术设计方案，涵盖系统架构、功能模块、数据模型及关键技术实现，作为开发团队的技术依据和项目管理的基准文档。

### 1.2 项目背景
现代都市人群面临饮食不规律、营养失衡、健康管理缺失等问题。传统饮食记录工具交互繁琐，缺乏智能化分析能力。本系统基于**React Native跨平台技术**与**云端AI大模型**，构建"记录-分析-规划-咨询"全链路膳食管理生态，通过多模态交互（拍照/语音/文字）降低使用门槛，利用AI实现个性化营养建议与智能食谱规划。

### 1.3 术语定义
- **BMR**：基础代谢率（Basal Metabolic Rate）
- **RAG**：检索增强生成（Retrieval-Augmented Generation）
- **VL模型**：视觉语言模型（Vision-Language Model）
- **OSS**：对象存储服务（Object Storage Service）
- **SSE**：服务器推送事件（Server-Sent Events）

### 2.2 技术选型矩阵

| 层级 | 技术方案 | 选型理由 |
|------|---------|---------|
| **移动端** | React Native 0.73 + Expo SDK 50 | 跨平台开发，Expo提供Camera/Storage等原生能力封装，无需配置Android Studio |
| **后端框架** | NestJS 10 (TypeScript) | 装饰器语法规范，内置依赖注入，模块化架构清晰，适合复杂业务分层 |
| **数据库** | PostgreSQL 15 + pgvector | JSONB存储AI非结构化数据，pgvector支持RAG向量检索 |
| **缓存** | Redis 7 | 会话存储、AI结果缓存（TTL 1h）、分布式锁（防止并发修改食谱） |
| **对象存储** | MinIO | 兼容S3 API，本地Docker部署方便，支持预签名URL安全访问 |
| **AI服务** | DashScope(qwen-vl/turbo) + Moonshot(kimi) | 双厂商冗余，VL模型负责图像识别，文本模型分工处理分析与规划 |
| **消息推送** | Expo Notifications | 跨平台推送，支持定时提醒（早餐/午餐/晚餐/饮水） |

### 3.2 详细功能设计

#### 3.2.1 首页模块（Home）
**核心功能**：数据 dashboard 与 AI 建议入口

- **营养环形图**：SVG动态渲染，展示热量/蛋白质/碳水/脂肪摄入进度（目标值 vs 实际值），支持点击跳转详情
- **AI智能建议卡片**：
  - 双模式展示：若无用户自定义提示，展示AI生成的营养建议；若有，随机展示用户自定义提示
  - 支持"刷新"按钮更换建议内容
  - 快捷操作按钮："咨询AI"、"查看详情"
- **今日记录时间线**：垂直时间轴展示当日已记录餐次（图标+食物名+热量+时间戳）
- **快捷记录入口**：悬浮按钮(FAB)展开拍照/语音/手动三模式

#### 3.2.2 记录模块（Record）
**核心功能**：多模态数据录入

**A. 拍照识菜模式**
- 调用Expo Camera组件，支持实时预览与拍照
- 图片上传至MinIO（预签名URL直传，减轻后端带宽）
- 异步任务队列（Bull）处理：
  1. 调用qwen-vl-plus识别菜名
  2. 调用qwen-turbo分析营养成分（热量/蛋白质/碳水/脂肪/钠）
  3. 结果存入PostgreSQL并缓存至Redis（Key: `ai:analyze:{imageHash}`, TTL: 1h）
- 前端轮询或SSE推送结果，用户确认份量（小/中/大份滑动条，对应0.5x/1.0x/1.5x系数）后保存

**B. 语音速记模式**
- 长按录音（React Native TouchableOpacity）
- 使用expo-av录制音频，格式AAC，上传后端
- 后端调用Whisper API或百度语音API转文字
- NLP解析（正则+词典匹配）提取食物实体，自动填充表单

**C. 手动录入模式**
- 搜索框支持模糊搜索（PostgreSQL全文检索或Elasticsearch）
- "最近常吃"快捷标签（基于用户历史记录Top 10）
- 分类浏览（主食/肉类/蔬菜/水果/饮品）
- 份量选择器：滑动条（100g-500g）+ 快捷按钮（小份/中份/大份）

#### 3.2.3 分析模块（Analytics）
**核心功能**：多维度数据可视化

**A. 今日视图（Day View）**
- 顶部统计卡片：已摄入/剩余额度/餐次记录/健康评分
- 营养成分详细分析：5项指标（蛋白质、碳水、脂肪、膳食纤维、维生素C）进度条+具体数值+食物来源
- 餐次记录详情：每餐的营养素拆解（蛋白质/碳水/脂肪）

**B. 本周视图（Week View）**
- 营养均衡度雷达图（5维）：基于本周平均数据生成
- 每日热量趋势柱状图（7天）：点击某天可查看当日详情
- 本周AI分析总结：自动生成本周饮食评价与改进建议

**C. 本月视图（Month View）**
- 每周热量趋势（5周均值）
- 月度统计数据：平均日摄入/月健康评分/合规天数
- 月度AI分析报告

**D. 历史视图（History）**
- 时间导航器：按月/周/日三级下钻
- 历史数据卡片列表：展示往期记录概览

#### 3.2.4 智能食谱模块（Meal Plan）
**核心功能**：个性化饮食规划

**A. 我的食谱（自定义模式）**
- **基础设置**：
  - 每日摄入量滑块（1200-3500 kcal，步长50）
  - 每日餐数选择（2/3/4/5餐，Segment Control）
  - 饮食目标单选（减脂/增肌/维持）
  - 口味偏好多选（清淡/微辣/无辣不欢/少油/低糖/高蛋白）
- **智能定制入口**：跳转至AI定制页面
- **食谱预览**：生成一周7天的食谱概览（早餐/午餐/晚餐/加餐），展示每餐热量与食材

**B. AI智能定制**
- 基于用户身体数据（身高/体重/年龄/性别/BMR）自动计算推荐摄入量
- AI生成详细食谱（调用kimi-v1长文本模型）：
  - 每日三餐具体菜品
  - 食材清单与份量
  - 烹饪建议
  - 营养分析（确保蛋白质/碳水/脂肪比例合理）
- 支持"重新生成"与"微调"

#### 3.2.5 AI咨询模块（Chat）
**核心功能**：营养问答与数据联动

- **首页数据快照**：聊天界面顶部固定展示今日摄入/剩余额度/餐次记录，实现上下文感知
- **对话界面**：类微信气泡，支持Markdown渲染（营养数据加粗显示）
- **快捷问题**：根据今日数据动态推荐（如"今天还能吃多少？"、"推荐今晚晚餐"）+ 通用问题（如"孕妇能吃螃蟹吗？"）
- **多轮对话**：支持上下文记忆（如"刚才说的那个食谱热量太高，换个低卡的"）
- **底部功能栏**：拍照识菜快捷入口、查看报告、历史记录

#### 3.2.6 提醒系统模块（Notification）
**核心功能**：全时段饮食行为干预

- **主开关**：总控所有提醒的启用/禁用
- **餐次提醒**：
  - 早餐提醒（默认07:30，可设07:00-09:00）
  - 午餐提醒（默认12:00，可设11:30-13:00）
  - 晚餐提醒（默认18:00，可设17:30-19:30）
- **饮水提醒**：
  - 间隔设置（1小时/2小时/3小时/4小时）
  - 运行时段（08:00-22:00，避免夜间打扰）
- **记录提醒**：用餐后30分钟自动提醒记录（基于用户活跃时间智能推算）
- **睡前提醒**：21:30提醒复盘当日饮食并预览明日食谱
- **提醒方式**：通知栏推送 + 应用内弹窗 + 震动（可独立开关）

#### 3.2.7 个性化提示库模块（Tip Library）
**核心功能**：用户自定义首页展示内容

- **提示列表管理**：展示已添加的自定义提示（边框颜色区分，如粉色/绿色）
- **添加提示**：文本框输入（限制100字），支持预设建议快速添加（如"记得多喝水"、"细嚼慢咽"）
- **展示策略**：若用户添加了自定义提示，首页随机展示用户提示；否则展示AI建议
- **编辑与删除**：左滑或点击按钮操作

#### 3.2.8 个人画像模块（Profile）
**核心功能**：用户数字身份管理

- **头像设置**：表情符号选择器（12个预设Emoji），点击切换
- **基础信息**：昵称、用户ID（只读）、个性签名
- **身体数据**：身高、体重、性别、年龄（用于计算BMR和每日推荐摄入量）
- **饮食画像标签**：AI根据历史记录生成的标签（如"早餐战士"、"低碳先锋"）
