## 2. 接口设计

### 2.1 基础规范

| 项 | 说明 |
|----|------|
| Base URL | `https://api.dietwise.cn/v1` |
| 协议 | HTTPS, TLS 1.2+ |
| 认证 | Bearer JWT Token (Authorization Header)<br>Access Token 7天, Refresh Token 30天 |
| Content-Type | `application/json` (请求体), `multipart/form-data` (文件上传) |
| 时间格式 | ISO 8601 + 时区: `2025-01-15T08:30:00+08:00` |
| 分页 | Cursor-based: `cursor` (上一页末条ID) + `limit` (默认20, 最大100) |
| 限流 | 普通接口 60次/min, AI接口 20次/min, 认证接口 10次/min |

### 2.2 统一响应格式

**成功**:
```json
{
  "code": 0,
  "message": "success",
  "data": { ... },
  "timestamp": "2025-01-15T08:30:00+08:00",
  "requestId": "req_abc123"
}
```

**错误**:
```json
{
  "code": 40100,
  "message": "Token已过期，请重新登录",
  "error": "TOKEN_EXPIRED",
  "data": null,
  "timestamp": "..."
}
```

### 2.3 错误码

| HTTP | 业务码 | 说明 |
|------|--------|------|
| 200 | 0 | 成功 |
| 400 | 40001 | 参数缺失或格式错误 |
| 400 | 40002 | 参数值不符合业务规则 |
| 401 | 40100 | Token缺失 |
| 401 | 40101 | Token已过期 |
| 401 | 40102 | Token签名无效 |
| 403 | 40300 | 无权限 |
| 404 | 40400 | 资源不存在 |
| 409 | 40900 | 资源冲突（如手机号重复） |
| 409 | 40901 | 数据同步冲突 |
| 429 | 42900 | 频率限制 |
| 500 | 50000 | 服务器内部错误 |
| 503 | 50300 | AI服务不可用 |
